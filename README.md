# TrustMeBro

## Host and serve the model (vLLM + ngrok)

Use **two terminals**. vLLM must be listening before you start ngrok.

**Terminal 1 — vLLM**

```bash
./serve_vllm.sh
```

Optional: `./serve_vllm.sh "microsoft/Phi-3-mini-4k-instruct" 8000`

**Terminal 2 — ngrok** (same port as vLLM, default `8000`)

```bash
./serve_ngrok.sh
```

Optional: `./serve_ngrok.sh 8000`

The OpenAI-compatible API is at `http://localhost:8000/v1/...` (or your ngrok HTTPS URL).

---

## Run the evaluation app

The evaluator is a FastAPI service that calls your model’s chat endpoint and scores MMLU (`abstract_algebra` validation).

**1. Install dependencies with [uv](https://docs.astral.sh/uv/)**

```bash
uv venv
source .venv/bin/activate
uv pip install -r requirements.txt
```

On Windows, activate with `.venv\Scripts\activate`.

**2. Start the API** (default port **8089**)

```bash
source .venv/bin/activate
python main.py
```

Or: `uvicorn main:app --host 0.0.0.0 --port 8089`

## Call `POST /evaluate` with cURL

Point `api_url` at your vLLM base URL (local or ngrok). Use the same `model_name` vLLM is serving.

**Local vLLM**

```bash
curl -sS -X POST "http://localhost:8089/evaluate" \
  -H "Content-Type: application/json" \
  -d '{
    "api_url": "http://localhost:8000",
    "model_name": "microsoft/Phi-3-mini-4k-instruct",
    "use_structured_output": true
  }'
```

**Through ngrok** (replace with your tunnel URL)

```bash
curl -sS -X POST "http://localhost:8089/evaluate" \
  -H "Content-Type: application/json" \
  -d '{
    "api_url": "https://YOUR-SUBDOMAIN.ngrok-free.dev",
    "model_name": "microsoft/Phi-3-mini-4k-instruct",
    "use_structured_output": true
  }'
```

If your inference server does not support structured JSON output, set `"use_structured_output": false`.

## JSON shapes

### Request body (`POST /evaluate`)

| Field | Type | Required | Description |
|--------|------|----------|-------------|
| `api_url` | string | yes | Base URL of the inference server (e.g. `http://localhost:8000` or ngrok origin). `/v1/chat/completions` is appended automatically if missing. |
| `model_name` | string | yes | Model id passed to the chat API (must match what vLLM is serving). |
| `use_structured_output` | boolean | no (default `true`) | If `true`, sends a JSON-schema `response_format` so the model replies with `{"letter":"A"\|"B"\|"C"\|"D"}`. Set `false` if your server rejects that. |

Example:

```json
{
  "api_url": "http://localhost:8000",
  "model_name": "microsoft/Phi-3-mini-4k-instruct",
  "use_structured_output": true
}
```

### Response body (`200 OK`)

Top-level fields:

| Field | Type | Description |
|--------|------|-------------|
| `accuracy` | number | Percent correct, **0–100**. |
| `correct` | integer | Number of items scored correct. |
| `total` | integer | Number of validation items (dataset size). |
| `details` | array | One object per validation row (see below). |

Example (values are illustrative):

```json
{
  "accuracy": 63.63636363636363,
  "correct": 7,
  "total": 11,
  "details": [
    {
      "question": "The cyclic subgroup of Z_24 generated by 18 has order",
      "gold_index": 0,
      "predicted_index": 0,
      "correct": true,
      "raw_reply": "{\"letter\":\"A\"}"
    }
  ]
}
```

Each element of `details` is either:

**Normal row** (model reply was parsed)

| Field | Type | Description |
|--------|------|-------------|
| `question` | string | MMLU question stem (full text). |
| `gold_index` | integer | Correct option index: `0` = A … `3` = D. |
| `predicted_index` | integer or null | Parsed prediction (`0`–`3`), or `null` if parsing failed. |
| `correct` | boolean | `true` if `predicted_index === gold_index`. |
| `raw_reply` | string | First 500 characters of the assistant `content` from the chat API. |

**Error row** (bad or missing chat response)

| Field | Type | Description |
|--------|------|-------------|
| `question` | string | Same as above. |
| `gold_index` | integer | Same as above. |
| `predicted_index` | null | Always `null` here. |
| `correct` | boolean | `false`. |
| `error` | string | Short error, e.g. `bad_response: ...`. |

Example error entry:

```json
{
  "question": "…",
  "gold_index": 2,
  "predicted_index": null,
  "correct": false,
  "error": "bad_response: …"
}
```
